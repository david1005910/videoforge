# VideoForge 사용자 매뉴얼

> macOS 전용 AI 영상 제작 스튜디오 — HBAS YouTube 채널을 위한 올인원 도구.

---

## 1. 설치 및 첫 실행

### 시스템 요구사항

- macOS 11.0 (Big Sur) 이상
- Intel x86_64 또는 Apple Silicon (Universal2 바이너리)
- 최소 8GB RAM (16GB 권장)
- 약 500MB 디스크 여유 공간

### 설치

1. GitHub Releases 페이지 또는 CI Artifacts에서 `.dmg` 파일을 다운로드합니다.
2. DMG 파일을 열고 VideoForge 아이콘을 Applications 폴더로 드래그합니다.
3. **Gatekeeper 해제** (Apple Developer ID 미서명 앱이므로 필수):

**방법 A — 우클릭 열기 (권장, 1회만):**

1. Finder에서 `/Applications/VideoForge.app` 우클릭
2. "열기" 클릭
3. "열기" 확인 팝업에서 다시 "열기" 클릭
4. 이후부터는 더블클릭으로 정상 실행됨

**방법 B — 터미널 (한 줄):**

```bash
xattr -cr /Applications/VideoForge.app
```

**방법 C — 시스템 설정:**

1. VideoForge를 더블클릭 (차단 팝업 나옴)
2. 시스템 설정 → 개인 정보 보호 및 보안 → 하단 "확인 없이 열기" 클릭

> 이 과정은 Apple Developer ID 서명이 없는 앱에서 1회만 필요합니다. 향후 공식 서명 버전이 배포되면 이 단계는 불필요합니다.

### 온보딩 (첫 실행)

첫 실행 시 온보딩 화면이 나타납니다:

1. **Welcome** — 앱 소개
2. **API Key 설정** — Gemini API 키를 입력합니다 (AI 채팅, TTS에 필요)
3. **완료** — 프로젝트 목록 화면으로 이동

> API 키는 macOS Keychain에 암호화 저장되며, 나중에 설정 페이지에서 변경할 수 있습니다.

## 2. 프로젝트 관리

### 프로젝트 목록 (`/`)

- **새 프로젝트**: 우측 상단 "새 프로젝트" 버튼 클릭
- **검색**: 상단 검색창에서 프로젝트 이름으로 필터링
- **열기**: 프로젝트 카드 클릭
- **삭제**: 프로젝트 카드의 삭제 버튼 (휴지통으로 이동)

### 새 프로젝트 만들기

1. **프로젝트 제목** 입력
2. **언어** 선택 (한국어, English, 日本語, 中文)
3. **해상도** 선택 (FHD 1920×1080, HD 1280×720, 4K 3840×2160, Shorts 1080×1920)
4. "만들기" 클릭 → 에디터로 이동

### 프로젝트 저장 위치

기본: `~/Documents/VideoForge/Projects/`
설정에서 변경 가능합니다.

## 3. 에디터

에디터는 3-패널 구조입니다:

| 패널            | 위치 | 기능                                   |
| --------------- | ---- | -------------------------------------- |
| 씬 목록         | 좌측 | 씬 추가/삭제/선택, 드래그로 순서 변경  |
| 스크립트 에디터 | 중앙 | 나레이션 스크립트 작성 (한국어 + 원문) |
| 인스펙터        | 우측 | 선택된 씬의 에셋 상태 및 프롬프트 확인 |

하단에는 **타임라인**이 있습니다:

- 초 단위 눈금자 표시
- 0.5x ~ 3x 줌 조절
- 씬 블록 드래그로 순서 재배치

### 씬 관리

- **씬 추가**: 좌측 패널 상단 "+" 버튼
- **씬 삭제**: 씬 우측 휴지통 아이콘 (마우스 호버 시 표시)
- **에셋 상태 아이콘**: 이미지(녹색), 나레이션(파란색), 자막(황색) — 각 에셋이 있으면 표시

### 스크립트 작성

- **주 스크립트**: 프로젝트 언어에 따라 한국어 또는 원문이 상단에 배치
- **보조 스크립트**: 번역/원문이 하단에 배치
- **노트**: 연출 메모, 참고사항 기록
- 스크립트 편집 후 포커스를 벗어나면(blur) 자동 저장

### 내보내기

1. 타이틀바의 "Export" 버튼 클릭
2. **해상도** 선택: FHD / HD / 4K / Shorts
3. **코덱** 선택: H.264 (YouTube/SNS 권장) 또는 ProRes (편집용, 용량 큼)
4. **비트레이트** 선택: 4M ~ 20M
5. 출력 폴더 선택 후 "Export" 클릭
6. 진행률 바가 표시되며, 완료 시 출력 폴더가 자동으로 열립니다

## 4. TTS (텍스트 음성 합성)

### 프로바이더

| 프로바이더 | 특징                 | API 키 필요 |
| ---------- | -------------------- | ----------- |
| Edge TTS   | 무료, Microsoft 음성 | 아니오      |
| Google TTS | Wavenet 고품질 음성  | 예          |
| Gemini TTS | Google AI 음성       | 예          |

### 사용 방법

1. 상단에서 프로바이더(Edge/Google/Gemini) 선택
2. **보이스** 선택 (언어별 음성 목록)
3. **속도** 조절 (0.5x ~ 2.0x)
4. 텍스트 입력 (최대 50,000자)
5. "합성하기" 클릭
6. 완성된 음성의 파형이 표시됨
7. "재생"으로 미리듣기, "저장"으로 파일 다운로드

> 동일한 텍스트+보이스+속도 조합은 자동 캐시됩니다. 재요청 시 즉시 반환됩니다.

## 5. 자막 생성 (STT + 정렬)

### 워크플로우

1. 씬에 나레이션 오디오가 있어야 합니다 (TTS에서 생성)
2. 에디터 하단 자막 패널에서 **STT 프로바이더** 선택 (OpenAI Whisper / Gemini / Whisper Local)
3. **스타일 프리셋** 선택 (기본 / HBAS)
4. "자막 생성" 클릭
5. STT → 강제 정렬(Forced Alignment) → ASS 자막 자동 생성

### Whisper Local (로컬 STT)

클라우드 없이 로컬에서 음성 인식을 수행할 수 있습니다.

1. 설정 > Whisper 섹션에서 whisper.cpp 바이너리를 다운로드합니다
2. 원하는 모델을 다운로드합니다 (권장: `ggml-large-v3-turbo-q5_0`, 547MB)
3. 자막 패널에서 프로바이더를 "Whisper (로컬)"로 변경
4. API 키 없이 오프라인에서 사용 가능합니다

> 로컬 STT는 Intel Mac에서 5분 오디오 기준 2~3분 소요됩니다.

### 단어 타임라인 편집

생성 후 단어별 시작/끝 시간을 밀리초 단위로 수동 조정할 수 있습니다. 수정 시 ASS 자막이 즉시 재생성됩니다.

## 6. AI 영상 생성 (Grok Imagine)

### 사전 준비

1. Grok 계정이 필요합니다
2. "Login" 버튼으로 브라우저가 열리면, 수동으로 로그인합니다
3. 쿠키가 저장되어 다음부터 자동 유지

### 영상 생성

- **프롬프트 입력**: 텍스트 또는 이미지 기반 영상 생성
- **배치 모드**: 여러 프롬프트를 큐에 넣고 순차 처리
- **진행률 추적**: 각 항목의 상태(대기/진행/완료/실패)가 실시간 표시
- **결과 갤러리**: 생성된 영상을 미리보기 및 다운로드

> Grok의 이용 약관을 준수하세요. 자동화는 개인 사용 목적으로 제한됩니다.

## 7. AI 이미지 생성 (Whisk / ImageFX)

### Whisk

- Subject, Scene, Style 이미지를 업로드하여 참조 기반 이미지 생성
- Google 계정 로그인 필요 (수동)

### ImageFX

- 텍스트 프롬프트 기반 이미지 생성
- Google 계정 로그인 필요 (수동)

두 서비스 모두 생성된 이미지의 EXIF에 프롬프트가 기록됩니다.

## 8. AI 채팅 어시스턴트

세 가지 전문 채팅 어시스턴트를 제공합니다:

| 어시스턴트             | 용도                            |
| ---------------------- | ------------------------------- |
| **DNA Script**         | 영상 대본 작성 지원             |
| **Cinematography**     | 촬영/연출 조언                  |
| **Thumbnail Analysis** | 썸네일 클릭률 예측 및 개선 제안 |

### 사용 방법

1. 채팅 페이지에서 어시스턴트 선택
2. 메시지 입력 후 전송
3. Markdown 형식의 응답 (코드 블록 복사 지원)
4. "Clear" 버튼으로 대화 초기화

> 채팅 기록은 프로젝트별로 저장됩니다.

### 썸네일 분석

1. 썸네일 이미지를 업로드
2. AI가 분석하여 클릭률 점수, 강점, 약점, 개선 제안을 반환
3. 결과가 시각적으로 정리되어 표시

## 10. 설정

### API Keys

- Gemini API 키를 입력하고 "Save" 클릭
- macOS Keychain에 암호화 저장

### Whisper 모델 관리

- **바이너리**: whisper.cpp 실행 파일 다운로드 (최초 1회)
- **모델**: 12개 GGML 모델 중 선택하여 다운로드/삭제
- 권장 모델: `ggml-large-v3-turbo-q5_0` (547MB, 품질/크기 균형)
- 저장 위치: `~/Library/Application Support/VideoForge/whisper/`

### 자동 업데이트

- "Auto-check for updates" 토글로 시작 시 업데이트 자동 확인 활성화/비활성화

### 접근성

- 폰트 크기: Small (14px) / Normal (16px) / Large (18px)
- 키보드 내비게이션: Tab/Shift+Tab으로 컨트롤 이동, Enter/Space로 활성화

### 진단

- "Export Error Report" 버튼으로 로그, 설정, 시스템 정보를 수집하여 폴더로 내보내기
- 문제 해결 시 이 리포트를 첨부해주세요

## 12. Grok Bridge 익스텐션

Puppeteer 봇 탐지가 발생하는 경우, Chrome 익스텐션을 통해 사용자의 실제 브라우저에서 Grok 영상 생성을 수행할 수 있습니다.

### 작동 방식

1. VideoForge가 WebSocket 서버를 시작합니다 (포트 9821)
2. Chrome 익스텐션이 이 서버에 연결합니다
3. 앱에서 생성 요청을 보내면 익스텐션이 사용자의 Grok 탭에서 실행합니다
4. 진행률과 결과가 앱으로 전달됩니다

> Bridge 익스텐션은 Puppeteer의 대안으로, 봇 탐지를 완전히 우회합니다.

## 17. 문제 해결

### 앱이 시작되지 않음

- macOS 시스템 설정 > 보안 및 개인 정보 보호에서 앱을 허용했는지 확인
- Intel Mac에서 Rosetta 설치가 필요할 수 있습니다 (Apple Silicon → Intel 빌드 사용 시)

### TTS 생성 실패

- Edge TTS: 인터넷 연결 확인
- Google/Gemini TTS: 설정에서 API 키가 올바르게 입력되었는지 확인

### Grok/Whisk 로그인 문제

- "Login" 후 열린 브라우저 창에서 수동 로그인이 필요합니다
- 2FA(이중 인증)가 활성화된 계정은 수동 로그인 시 추가 단계가 필요합니다
- 쿠키가 만료되면 재로그인이 필요합니다

### 메모리 부족

- Intel 16GB 환경에서 장시간 사용 시 Vite dev server 메모리 누수가 발생할 수 있습니다
- 개발 모드에서 1~2시간마다 재시작을 권장합니다
- 프로덕션 빌드에서는 이 문제가 발생하지 않습니다

### 에러 리포트 내보내기

설정 > 진단 > "Export Error Report" 클릭으로 로그와 시스템 정보를 수집할 수 있습니다. 버그 리포트 시 이 파일을 첨부해주세요.
